1. 포탈 개요

BizOps 산출물 포탈은 Toomics Global BizOps팀의 모든 운영 산출물을 한 곳에서 열람할 수 있는 웹 포탈입니다.

💡 핵심 가치: 기존에 Confluence, Slack, Google Docs 등에 흩어져 있던 문서를 하나의 URL로 통합하여, 누구나 · 언제든 · 즉시 접근할 수 있게 만들었습니다.

2. 접속 방법

2.1 PC에서 접속

1

브라우저(Chrome, Edge, Safari 등)를 엽니다.

2

주소창에 아래 URL을 입력하거나, 북마크에서 클릭합니다.

https://tg-bizops.github.io/BizOPS/
3

포탈 메인 페이지가 로드되면, 카테고리별 문서 목록이 표시됩니다.

✅ 팁: 자주 사용한다면 브라우저 북마크에 추가해두세요. Ctrl+D (Mac: ⌘+D)

2.2 모바일에서 접속

모바일 브라우저에서도 동일한 URL로 접속 가능합니다. 반응형 디자인이 적용되어 모바일 화면에 최적화됩니다.

⚠️ 참고: 일부 문서(Excel 파일 등)는 모바일에서 다운로드만 가능하고, PC에서 열어야 정상적으로 확인됩니다.

2.3 슬랙에서 바로 접속

슬랙에서 포탈 링크를 공유하면 미리보기가 표시됩니다. 링크를 클릭하면 바로 포탈로 이동합니다.

4. 문서 열람 및 활용

4.1 HTML 문서 열람

1

포탈 메인 페이지에서 원하는 카테고리의 문서 카드를 클릭합니다.

2

새 탭에서 문서가 열립니다. 문서 내에서 목차 링크를 통해 원하는 섹션으로 바로 이동할 수 있습니다.

3

문서 내 응대 템플릿은 드래그 → 복사하여 바로 사용할 수 있습니다.

✅ 팁: 문서 내에서 Ctrl+F (Mac: ⌘+F)로 특정 키워드를 검색할 수 있습니다.

4.2 PDF로 저장/인쇄

모든 HTML 문서는 @media print 스타일이 적용되어 있어, 브라우저 인쇄 기능으로 깔끔한 PDF를 만들 수 있습니다.

1

문서 페이지에서 Ctrl+P (Mac: ⌘+P)를 누릅니다.

2

대상을 "PDF로 저장"으로 선택합니다.

3

"배경 그래픽" 옵션을 체크하면 색상과 차트가 포함됩니다.

4.3 Excel 파일 다운로드

차지백 증빙 템플릿 등 Excel 파일은 클릭 시 자동 다운로드됩니다. 다운로드 후 로컬에서 편집하여 사용하세요.

⚠️ 주의: 다운로드한 Excel 파일을 수정해도 포탈의 원본에는 영향이 없습니다. 항상 포탈에서 최신 버전을 다운로드하여 사용하세요.

5.1 포탈 내 검색

포탈 메인 페이지 상단의 검색 바에 키워드를 입력하면, 문서 제목을 기준으로 실시간 필터링됩니다.

5.2 카테고리 필터

포탈 상단의 카테고리 탭을 클릭하면 해당 영역의 문서만 표시됩니다.

5.3 문서 내 검색

개별 문서를 열어 브라우저의 Ctrl+F로 문서 내용을 검색할 수 있습니다.

6. 문서별 활용 가이드

6.1 CS 운영 매뉴얼 활용

용도

CS 티켓 응대 시 카테고리별 템플릿을 복사하여 사용합니다.

활용 순서

  1. 티켓의 카테고리를 확인합니다 (구독/환불/계정/결제 등)
  2. 매뉴얼에서 해당 섹션으로 이동합니다 (목차 클릭)
  3. 적합한 응대 템플릿을 찾아 복사합니다
  4. 고객 정보에 맞게 빈칸(_____)을 수정하여 회신합니다
💡 참고: EN 매뉴얼과 KO 매뉴얼이 별도로 제공됩니다. 마켓 언어에 맞는 매뉴얼을 사용하세요.

6.2 차지백 소명 업무

활용 순서

  1. 차지백 운영 매뉴얼에서 PG사별 소명 프로세스를 확인합니다
  2. 차지백 플로우 다이어그램에서 전체 흐름을 파악합니다
  3. 소명 폼 V5를 다운로드하여 내용을 작성합니다
  4. 증빙 템플릿 v2를 활용하여 증빙 자료를 정리합니다
  5. PG사에 소명서를 제출합니다

7. 자주 묻는 질문 (FAQ)

Q1. 포탈에 접속하려면 로그인이 필요한가요?

아닙니다. GitHub Pages 기반 퍼블릭 사이트로, URL만 알면 누구나 접근 가능합니다.

Q2. 문서가 업데이트되면 어떻게 알 수 있나요?

포탈 메인 페이지의 "최종 업데이트" 날짜를 확인하세요. 주요 업데이트 시 슬랙으로 공지합니다.

Q3. 특정 문서의 이전 버전을 볼 수 있나요?

Git 기반으로 모든 변경 이력이 보존됩니다. 이전 버전이 필요한 경우 팀 리드에게 요청하세요.

Q4. 문서에 오류가 있으면 어떻게 하나요?

팀 리드 또는 슬랙 #bizops 채널에 알려주세요. 확인 후 수정하여 재배포합니다.

Q5. 새로운 문서를 요청하고 싶어요.

아래 섹션 8. 문서 요청 및 피드백을 참고하세요.

Q6. 오프라인에서도 볼 수 있나요?

인터넷 연결이 필요합니다. 오프라인 사용이 필요한 경우 PDF로 저장해두세요 (섹션 4.2 참고).

Q7. 모바일에서 문서가 깨져 보여요.

대부분의 문서는 반응형이지만, 일부 표/차트가 포함된 문서는 PC 브라우저에서 최적화되어 있습니다. 가로 모드로 전환하거나 PC를 이용해주세요.

8. 문서 요청 및 피드백

8.1 신규 문서 요청

새로운 산출물이 필요한 경우 아래 정보를 포함하여 팀 리드에게 요청하세요:

  • 문서 제목 (예: "DE 마켓 특이 케이스 대응 가이드")
  • 카테고리 (CS/차지백/AI/FAQ/이용약관/주간리포트/IP/기타)
  • 목적 및 대상 (누가, 언제, 왜 사용하는지)
  • 우선순위 (높음/보통/낮음)

8.2 기존 문서 수정 요청

문서 내용이 현행과 맞지 않거나 개선이 필요한 경우:

  • 해당 문서 이름과 수정이 필요한 부분을 구체적으로 알려주세요
  • 슬랙 DM 또는 #bizops 채널 모두 가능합니다
