1. 포탈 개요

BizOps 산출물 포탈은 Toomics Global BizOps팀의 모든 운영 산출물을 한 곳에서 열람할 수 있는 웹 포탈입니다.

핵심 가치: 기존에 Confluence, Slack, Google Docs 등에 흩어져 있던 문서를 하나의 URL로 통합하여, 누구나 · 언제든 · 즉시 접근할 수 있게 만들었습니다.

2. 접속 방법

2.1 PC에서 접속

1

브라우저(Chrome, Edge, Safari 등)를 엽니다.

2

주소창에 아래 URL을 입력하거나, 북마크에서 클릭합니다.

https://tg-bizops.github.io/BizOPS/
3

포탈 메인 페이지가 로드되면, 카테고리별 아코디언 목록이 표시됩니다.

팁: 자주 사용한다면 브라우저 북마크에 추가해두세요. Ctrl+D (Mac: Cmd+D)

2.2 모바일에서 접속

모바일 브라우저에서도 동일한 URL로 접속 가능합니다. 반응형 디자인이 적용되어 모바일 화면에 최적화됩니다.

참고: 일부 문서(Excel 파일 등)는 모바일에서 다운로드만 가능하고, PC에서 열어야 정상적으로 확인됩니다.

2.3 슬랙에서 바로 접속

슬랙에서 포탈 링크를 공유하면 미리보기가 표시됩니다. 링크를 클릭하면 바로 포탈로 이동합니다.

4. 문서 열람 및 활용

4.1 HTML 문서 열람

1

포탈 메인 페이지에서 원하는 카테고리 아코디언을 클릭하여 펼칩니다.

2

문서 카드를 클릭하면 해당 문서가 열립니다. 문서 내에서 목차 링크를 통해 원하는 섹션으로 바로 이동할 수 있습니다.

3

문서 내 응대 템플릿은 드래그 → 복사하여 바로 사용할 수 있습니다.

팁: 문서 내에서 Ctrl+F (Mac: Cmd+F)로 특정 키워드를 검색할 수 있습니다.

4.2 PDF로 저장/인쇄

모든 HTML 문서는 @media print 스타일이 적용되어 있어, 브라우저 인쇄 기능으로 깔끔한 PDF를 만들 수 있습니다.

1

문서 페이지에서 Ctrl+P (Mac: Cmd+P)를 누릅니다.

2

대상을 "PDF로 저장"으로 선택합니다.

3

"배경 그래픽" 옵션을 체크하면 색상과 차트가 포함됩니다.

4.3 Excel 파일 다운로드

차지백 소명 폼 등 Excel 파일은 클릭 시 자동 다운로드됩니다. 다운로드 후 로컬에서 편집하여 사용하세요.

주의: 다운로드한 Excel 파일을 수정해도 포탈의 원본에는 영향이 없습니다. 항상 포탈에서 최신 버전을 다운로드하여 사용하세요.

5. 검색 및 필터링

5.1 통합 검색

포탈 메인 페이지의 검색 바에 키워드를 입력하면, 문서 제목·설명·배지를 기준으로 실시간 필터링됩니다. 검색 결과 건수가 검색바 하단에 표시됩니다.

검색 예시결과
차지백차지백 관련 문서 필터링
chatbotAI 챗봇 관련 문서 필터링
manual매뉴얼 문서 전체 필터링
v2v2 버전 문서 필터링

5.2 카테고리 필터 필

검색 바 아래의 필터 필(pill) 버튼을 클릭하면 해당 카테고리의 문서만 표시됩니다. "전체"를 클릭하면 필터가 해제됩니다.

필터 필표시 문서
CSCS 운영 카테고리 8건
AIAI 챗봇 카테고리 13건
CB차지백 카테고리 4건
IPIP 보호 카테고리 1건
TK대시보드 카테고리 2건
기타기타 카테고리 3건
검색 + 필터 조합: 카테고리 필터를 선택한 상태에서 키워드를 입력하면, 해당 카테고리 내에서만 검색됩니다.

5.3 문서 내 검색

개별 문서를 열어 브라우저의 Ctrl+F로 문서 내용을 검색할 수 있습니다.

6. 버전 관리

6.1 버전 배지

모든 문서 카드에는 버전 배지가 표시됩니다. 문서의 현재 버전을 한눈에 확인할 수 있습니다.

표시 위치형태설명
문서 카드 우측v2.1현재 문서의 버전 번호
카드 우측 끝Manual문서 유형 배지 (Manual, Analysis, Flow, Template 등)

6.2 버전 히스토리

버전 이력이 있는 문서의 카드 우측에 시계 아이콘 버튼이 표시됩니다. 클릭하면 해당 문서의 버전별 변경 이력이 드롭다운으로 표시됩니다.

1

문서 카드 우측의 시계 버튼을 클릭합니다.

2

버전별 날짜와 변경 요약이 카드 하단에 드롭다운으로 표시됩니다.

3

다시 시계 버튼을 클릭하거나 다른 문서의 히스토리를 열면 자동으로 닫힙니다.

버전 히스토리가 있는 주요 문서

문서현재 버전이력 건수
CS Operations Manual (EN/KO)v2.12건
차지백 운영 매뉴얼v1.33건
Chargeback Explanation Formv5.04건

6.3 변경 히스토리

포탈 하단의 "변경 히스토리" 영역을 클릭하면 포탈 전체의 변경 이력을 확인할 수 있습니다. 날짜별 주요 변경사항이 최신순으로 표시됩니다.

7. 문서별 활용 가이드

7.1 CS 운영 매뉴얼 활용

용도

CS 티켓 응대 시 카테고리별 템플릿을 복사하여 사용합니다.

활용 순서

  1. 티켓의 카테고리를 확인합니다 (구독/환불/계정/결제 등)
  2. 매뉴얼에서 해당 섹션으로 이동합니다 (목차 클릭)
  3. 적합한 응대 템플릿을 찾아 복사합니다
  4. 고객 정보에 맞게 빈칸(_____)을 수정하여 회신합니다
참고: EN 매뉴얼과 KO 매뉴얼이 별도로 제공됩니다. 마켓 언어에 맞는 매뉴얼을 사용하세요.

7.2 차지백 소명 업무

활용 순서

  1. 차지백 운영 매뉴얼에서 PG사별 소명 프로세스를 확인합니다
  2. 차지백 플로우 다이어그램에서 전체 흐름을 파악합니다
  3. Explanation Form v5를 다운로드하여 내용을 작성합니다
  4. 증빙 템플릿 미리보기를 참고하여 증빙 자료를 정리합니다
  5. PG사에 소명서를 제출합니다

7.4 CS Ticket Dashboard

대시보드는 인터랙티브 HTML로, 필터/차트를 직접 조작하여 데이터를 확인할 수 있습니다.

상세 사용법은 대시보드 운영 매뉴얼을 참고하세요.

7.5 주간 분석 리포트

기타 카테고리의 "CS + 차지백 주간 분석 리포트"를 클릭하면 주간 리포트 아카이브 페이지로 이동합니다. 각 주차별로 상세 분석본과 Executive Summary 2종이 제공됩니다.

8. 문서 업데이트 요청

포탈 메인 페이지 하단에 문서 업데이트 요청 시스템이 내장되어 있습니다. 슬랙 DM이나 구두 요청 대신, 포탈에서 직접 요청을 제출하고 처리 현황을 확인할 수 있습니다.

8.1 요청 현황 확인

포탈 하단의 "요청 현황" 섹션에서 현재 접수된 요청의 상태를 확인할 수 있습니다.

항목설명
상태 배지대기중 진행중 완료 — 상태별 건수 표시
요청 테이블최근 5건의 요청 정보 (ID, 요청자, 대상문서, 유형, 우선순위, 상태, 날짜)
자동 새로고침페이지 로드 시 자동으로 최신 데이터를 가져옵니다

8.2 업데이트 요청 제출

"문서 업데이트 요청" 섹션의 폼을 작성하여 요청을 제출합니다.

1

요청자 이름을 입력합니다.

2

대상 문서를 드롭다운에서 선택합니다. 포탈에 등록된 모든 문서가 목록에 자동으로 표시됩니다.

3

요청 유형을 선택합니다.

유형설명
오류 수정문서 내 잘못된 정보, 오타, 링크 오류 등
내용 추가새로운 내용, 섹션, 템플릿 등 추가 요청
내용 삭제더 이상 유효하지 않은 내용 삭제 요청
버전 업데이트현행 프로세스 변경에 따른 전체 갱신
기타위에 해당하지 않는 기타 요청
4

요청 내용을 상세히 작성합니다. 어떤 부분을 어떻게 수정해야 하는지 구체적으로 기재하세요.

5

우선순위를 선택합니다.

  • 높음 — 업무에 즉시 영향을 미치는 긴급한 수정
  • 보통 — 일반적인 업데이트 (기본값)
  • 낮음 — 개선 사항, 편의성 관련
6

"요청 제출" 버튼을 클릭합니다. 성공 시 토스트 메시지가 표시되고 폼이 초기화됩니다.

자동 처리: 제출된 요청은 Google Sheets에 자동 기록되고, 슬랙으로 알림이 전송됩니다. 요청 현황도 자동으로 새로고침됩니다.

8.3 요청 처리 흐름

단계상태설명

9. 자주 묻는 질문 (FAQ)

Q1. 포탈에 접속하려면 로그인이 필요한가요?

아닙니다. GitHub Pages 기반 퍼블릭 사이트로, URL만 알면 누구나 접근 가능합니다.

Q2. 문서가 업데이트되면 어떻게 알 수 있나요?

문서 카드에 Updated 배지가 표시됩니다. 주요 업데이트 시 슬랙으로도 공지합니다.

Q3. 특정 문서의 이전 버전을 볼 수 있나요?

문서 카드의 시계 버튼을 클릭하면 버전 히스토리를 확인할 수 있습니다. 실제 이전 버전 파일이 필요한 경우 팀 리드에게 요청하세요 (Git 기반으로 모든 이력 보존).

Q4. 문서에 오류가 있으면 어떻게 하나요?

포탈 하단의 "문서 업데이트 요청" 폼에서 직접 수정 요청을 제출할 수 있습니다. 요청 유형에서 "오류 수정"을 선택하세요.

Q5. 새로운 문서를 요청하고 싶어요.

포탈의 업데이트 요청 폼에서 요청 유형 "내용 추가"를 선택하고, 요청 내용에 신규 문서에 대한 정보(제목, 카테고리, 목적, 대상)를 작성하여 제출하세요.

Q6. 오프라인에서도 볼 수 있나요?

인터넷 연결이 필요합니다. 오프라인 사용이 필요한 경우 PDF로 저장해두세요 (섹션 4.2 참고).

Q7. 모바일에서 문서가 깨져 보여요.

대부분의 문서는 반응형이지만, 일부 표/차트가 포함된 문서는 PC 브라우저에서 최적화되어 있습니다. 가로 모드로 전환하거나 PC를 이용해주세요.

Q8. 업데이트 요청을 제출했는데 현황판에 안 보여요.

페이지를 새로고침하면 최신 데이터가 로드됩니다. 제출 직후에는 자동으로 현황판이 갱신되지만, 네트워크 상황에 따라 잠시 지연될 수 있습니다.

Q9. 포탈 매뉴얼은 어디서 볼 수 있나요?

포탈 메인 페이지 상단 히어로 영역의 "포탈 매뉴얼" 바로가기 버튼을 클릭하면 이 문서로 이동합니다.
