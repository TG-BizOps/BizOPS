핵심 기능

AI Draft 생성

Compose 영역의 "✨ Generate AI Draft" 버튼을 클릭하면 AI가 자동으로 답변 초안을 생성합니다.

✨ Generate AI Draft 클릭
→ 스팸 체크 → RAG(매뉴얼 매칭) → 과거 사례 검색 → Claude AI 호출
→ 품질 채점 → 운영자 확인 체크 → Draft 표시

95/5 규칙

AI는 새로운 정보를 만들어내지 않습니다.
95% = CS 매뉴얼 + FAQ + 과거 상담원 답변에서 가져온 내용
5% = 인사말, 고객명, 연결어 등 형식적 조정만

생성 결과 3가지 경우

  • 정상 Draft가 textarea에 표시 → 확인/수정 후 발송
  • 스팸 감지 빨간 배너 표시 → Mark as Spam 또는 무시
  • NO_MATCH 매뉴얼에 매칭 없음 → 에스컬레이션 권장
  • 환불 결정 환불 요청 시 4개 버튼 표시 → 결정 후 재생성 (아래 참조)

환불 요청 처리 흐름 (Refund Followup)

고객이 환불을 요청하면 AI가 Template 3.1.1 (접수 확인 + 자격 조건 안내) 초안을 먼저 생성하고, 아래 4개 결정 버튼이 표시됩니다:

  1. 환불 승인 (Approved) → Template 3.2.1로 재생성
  2. 콘텐츠 접근 (Content Accessed) → Template 3.3.1로 재생성
  3. 14일 초과 (14-Day Exceeded) → Template 3.3.2로 재생성
  4. 코인 사용 (Bonus Coins Used) → Template 3.3.4로 재생성

💡 구독 해지 + 환불 복합 요청 시 Template 2.7.1이 자동 선택됩니다 (해지 확인 + 환불 심사 동시 안내).

5
특수 상황

스팸 / NO_MATCH / 운영자 확인 대응

⚠️ 스팸 감지 (빨간 배너)

11종 패턴(P1~P11)으로 자동 감지됩니다. AI Draft 생성이 차단되고 경고 배너가 표시됩니다.

패턴유형주요 감지 키워드
P1법률사무소 사칭"LLC", "DMCA notice", "evidence record"
P2랜덤 해시 문자열30자+ hex 문자열, 의미없는 긴 코드
P3외부 링크/다운로드.exe 파일, bit.ly, "click to download"
P4긴급성 압박"urgent action", "within 48 hours", "legal action"
P5파트너십 사칭"creator partnership", "exclusive offer"
P6가짜 구독 해제about:{LINK}, {UNSUBSCRIBE_LINK}
P7외부 서비스 홍보"CapCut Pro", "Netflix free"
P8외부 이메일 응답reply@non-toomics-domain.com
P9소셜미디어 사칭"Facebook policy violation" + "permanently disabled"
P10보안 사칭"penetration tester", "CVSS", "subdomain takeover" + 결제 요구
P11영업 스팸"SEO services", "app development", "increase traffic"

"Mark as Spam" 버튼 주의사항

클릭 시 확인 다이얼로그가 뜹니다. 확인하면:

  • 해당 티켓이 삭제됩니다
  • 요청자(고객) 계정이 정지됩니다
  • 되돌릴 수 없습니다 — 반드시 스팸이 확실할 때만 사용

⚠️ NO_MATCH (주황 배너)

매뉴얼/과거 사례에 매칭 없음

  • AI Draft가 생성되지 않습니다
  • "Escalate to Team Lead" 버튼으로 팀 리드에게 전달
  • 직접 답변 작성 후 Reply로 처리해도 됩니다

ℹ️ 운영자 확인 필요 (파란 배너)

Draft는 정상 생성되지만, 고객 주장 중 사실 확인이 필요한 항목이 감지되었습니다.

ID확인 항목감지 예시확인 방법
V1작품 서비스 상태"can't read purchased title"관리자 패널 → 해당 작품 상태 확인
V2결제 내역"charged twice", "unauthorized"관리자 패널 → 결제 이력 확인
V3계정 상태"account suspended", "can't login"관리자 패널 → 계정 상태/제재 이력
V4VIP 구독 상태"paid but no VIP", "subscription inactive"관리자 패널 → VIP 구독 상태

처리 흐름

파란 배너 확인 → 관리자 패널에서 사실 확인 → Draft 내용 수정 (필요시) → Reply & Pending 또는 Reply & Solved

품질 관리

품질 점수 확인 & 수정

AI Draft 생성 후 자동으로 품질 점수가 표시됩니다. 6가지 차원으로 평가됩니다.

차원가중치채점 기준
Template Match25%매뉴얼 템플릿 사용 여부
Safety20%금지 패턴 (경쟁사명, 잘못된 도메인, 미치환 변수)
Actionability20%실행 가능한 안내 포함 (링크, 절차 등)
Relevance15%고객 문의에 대한 정확한 응답
Completeness10%인사말 + 본문(80자+) + 맺음말
Length10%100~2,000자 최적

점수별 해석

80%+
Good

→ 바로 발송 가능. 내용 한번 훑어보고 Reply & Solved

60~79%
Fair

→ 일부 수정 필요. 특히 Safety 관련 항목 확인

<60%
Poor

→ 전면 재작성 권장. Draft 삭제 후 직접 작성하거나 에스컬레이션

Re-score 버튼

Draft를 수정한 후 "Re-score" 버튼을 클릭하면 수정된 내용으로 품질 점수를 재계산합니다. 수정 → Re-score → 점수 확인 → 발송 흐름을 권장합니다.

7
핵심 액션

답변 발송 — 4가지 액션

Draft 확인/수정 후 아래 4가지 버튼 중 하나를 선택합니다.

📨
Reply & Pending

공개 답변 발송 후 상태를 Pending(고객 응답 대기)으로 변경.

사용 시점: 고객에게 추가 정보를 요청할 때, 고객 확인이 필요할 때

✅
Reply & Solved

공개 답변 발송 후 상태를 Solved(해결 완료)로 변경.

사용 시점: 답변으로 문제가 해결되는 경우 (환불 안내, 방법 안내 등)

📄
Internal Note

내부 메모만 추가. 고객에게는 보이지 않음.

사용 시점: 팀 공유 메모, 진행 상황 기록, 내부 커뮤니케이션

🔺
Escalate

내부 메모 추가 + 상태를 In Progress로 변경 + "escalated" 태그 추가.

사용 시점: 팀 리드 확인 필요, 복잡한 환불/결제 분쟁, NO_MATCH

에스컬레이션 규칙

  • 에스컬레이션 시 사유를 반드시 Compose에 작성 (빈 메모 금지)
  • 팀 리드가 24시간 이내 확인 필요
  • 처리 완료 후 Reply & Solved로 마감
8
상태 관리

상태 직접 변경

Draft 없이 상태만 변경해야 할 때, 티켓 상세 화면 상단의 드롭다운 + Update 버튼을 사용합니다.

변경 가능 상태사용 시점
Open작업 시작을 명시, 또는 Pending에서 상담원이 추가 조치할 때
In Progress에스컬레이션 처리 중 (Escalate 버튼과 동일 효과)
Pending고객 응답 대기로 전환 (답변 없이 상태만 변경)
Solved문제 해결 완료 (답변 없이 종결 — 이미 다른 채널로 처리된 경우)

체크리스트

Do & Don't

✅ DO — 이렇게 하세요

  • ✓
    AI Draft를 반드시 읽어본 후 발송
  • ✓
    V1~V4 배너가 뜨면 관리자 패널에서 사실 확인 후 발송
  • ✓
    Draft 수정 후 Re-score로 품질 재확인
  • ✓
    환불/결제 민감 건은 팩트 확인 후 Reply
  • ✓
    확실한 스팸만 Mark as Spam
  • ✓
    모르겠으면 Escalate (사유 기재)
  • ✓
    고객 이름/이메일이 Draft에 정확히 들어갔는지 확인

❌ DON'T — 이것은 금지

  • ✕
    Draft를 읽지 않고 바로 발송
  • ✕
    스팸 의심 티켓에 답변 발송
  • ✕
    매뉴얼에 없는 환불 약속/할인 제공
  • ✕
    경쟁사(Webtoon, Tapas 등) 언급
  • ✕
    en.toomics.com 링크 사용 (마켓별 도메인 사용)
  • ✕
    에스컬레이션 시 사유 미기재
  • ✕
    품질 60% 미만 Draft를 수정 없이 발송

FAQ & 트러블슈팅

문제해결 방법
"Generate AI Draft" 버튼이 안 눌려요 티켓을 먼저 선택해야 합니다. 좌측 목록에서 티켓 클릭 후 시도
Draft 생성 중 "Loading..." 이 오래 걸려요 Claude AI 호출 중 (보통 3~8초). 30초 이상 소요 시 페이지 새로고침 후 재시도
갑자기 로그인 화면으로 돌아갔어요 세션 만료(8시간). 다시 로그인하면 됩니다
티켓 목록이 비어있어요 필터 확인 — Status/Market/Category가 "All"인지 체크. Search에 오타가 없는지 확인
Reply 발송 후 에러가 났어요 Zendesk 연결 문제일 수 있습니다. 1~2분 후 재시도. 반복 시 관리자에게 보고
Draft 내용이 영어인데 고객은 일본어예요 마켓 자동 감지가 작동합니다. 감지 실패 시 Draft를 수동으로 해당 언어로 수정
Mark as Spam을 실수로 눌렀어요 확인 다이얼로그에서 "취소" 가능. 이미 확인한 경우 Zendesk 관리자만 복구 가능 → 즉시 보고
품질 점수가 0%로 나와요 Safety 위반 (경쟁사명, 잘못된 URL 등). Draft 내용에서 해당 문구 삭제 후 Re-score

추가 도움이 필요할 때

  • 기술 문제: 관리자에게 Slack 또는 이메일로 문의
  • 정책 질문: CS AI Dashboard 운영 매뉴얼 참조 (상세 정책서 + API 레퍼런스 포함)
  • 전략 배경: AI CS 통합 로드맵 프레젠테이션 참조
