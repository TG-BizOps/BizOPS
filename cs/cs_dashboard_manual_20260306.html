1. System Overview

CS AI Dashboard는 투믹스(Toomics) 글로벌 플랫폼의 고객 지원 상담원을 위한 AI 보조 도구입니다. Zendesk 티켓 조회부터 AI 기반 답변 초안 생성, 품질 채점, 스팸 감지, 에스컬레이션까지 CS 워크플로우 전체를 단일 인터페이스에서 처리합니다.

1.1 핵심 기능

Ticket Management

Zendesk 티켓 실시간 조회, 상태별/마켓별/카테고리별 필터링, KPI 대시보드

AI Draft Generation

Claude AI + RAG 기반 다국어 답변 초안 자동 생성 (11개 언어 지원)

Quality Scoring

4차원 품질 점수 (템플릿 매칭, 완성도, 안전성, 길이) 실시간 채점

Spam/Phishing Detection

법률사무소 사칭, 해시 문자열, 외부 링크 등 9개 패턴 자동 감지

Verification Checker

고객 주장 중 운영자 확인 필요 항목 자동 감지 (V1~V4: 작품상태, 결제, 계정, VIP)

1.2 대상 사용자

역할접근 범위주요 기능
CS 상담원전체티켓 조회, AI Draft 생성, Reply & Pending/Solved, Escalate
팀 리드전체에스컬레이션 리뷰, 스팸 처리, KPI 모니터링

4. Authentication & Access Control

4.1 인증 방식

JWT (JSON Web Token) 기반 Stateless 인증을 사용합니다.

항목사양
알고리즘HS256 (HMAC-SHA256)
토큰 유효기간8시간
저장 위치브라우저 sessionStorage (탭 닫으면 삭제)
전송 방식Authorization: Bearer {token} 헤더

4.2 로그인 플로우

Username/Password 입력
→
POST /api/auth/login
→
JWT 토큰 발급
→
sessionStorage 저장

4.3 세션 만료 처리

API 호출 시 401 응답이 오면 자동으로 로그인 화면으로 리다이렉트됩니다. 토큰 만료(8시간) 후 재로그인이 필요합니다.

Policy 현재는 단일 계정(settings 기반)만 지원합니다. 다중 사용자가 필요한 경우 DB 기반 인증으로 확장이 필요합니다.

6.2 액션 버튼 (3개)

버튼유형동작Zendesk 상태사용 시점
Reply & Pending Public Reply 고객에게 공개 답변 발송 + 상태 변경 pending 추가 정보/확인이 필요할 때 (고객 응답 대기)
Reply & Solved Public Reply 고객에게 공개 답변 발송 + 상태 변경 solved 문의가 완전히 해결되었을 때
Escalate Internal Note 내부 메모 추가 + 상태 변경 + 태그 추가 In Progress (custom) + tag: escalated 팀 리드 확인이 필요한 복잡/민감 건

6.3 에스컬레이션 정책

Escalation Policy Escalate 버튼 클릭 시 티켓은 "In Progress" 상태 + "escalated" 태그가 부여됩니다. 팀 리드가 Zendesk 또는 대시보드에서 에스컬레이션된 티켓을 확인하고, 처리 방향을 결정한 뒤 답변합니다.

에스컬레이션 기준

  • 환불 금액 $100 이상 또는 다건 환불 요청
  • Fraud/악의적 이용 의심 건
  • 법적 조치 언급 또는 외부 기관 민원
  • 기술적 버그 (개발팀 확인 필요)
  • VIP 고객 또는 인플루언서 관련 민원

6.4 Status Change (수동)

상세 패널 상단의 Status 드롭다운에서 직접 상태를 변경할 수 있습니다: Open, In Progress, Pending, Solved.

6.5 대화 스레드

티켓의 모든 댓글이 시간순으로 표시됩니다.

유형표시색상
Customer고객 아이콘 + 이름파란색 (좌측 보더)
Agent상담원 아이콘초록색 (좌측 보더)
Internal Note이탤릭 + 노란 배경주황색 (좌측 보더)

7. AI Draft Generation

7.1 생성 플로우

1. Ticket 선택
→
2. Generate AI Draft 클릭
→
3. 스팸 체크
→
4. RAG 참조
→
5. Claude 생성
→
6. 품질 채점
→
7. 운영자 확인 감지

7.2 상세 단계

  1. Ticket Fetch — Zendesk API로 티켓 상세 + 댓글 스레드 조회
  2. Customer Text 추출 — 고객 최근 3개 공개 메시지를 결합
  3. Spam Check — 스팸/피싱 패턴 탐지 (§10 참조). 스팸이면 경고 배너 표시, Draft 미생성
  4. RAG Lookup — 티켓 카테고리 → CS 매뉴얼 섹션 매핑 → 최대 4,000자 컨텍스트 추출
  5. Claude Generation — System Prompt (매뉴얼 컨텍스트 + 다국어 인사/맺음말) + User Message (티켓 정보) → Claude Sonnet 호출
  6. Quality Score — 생성된 텍스트를 4차원 채점 (§9 참조)
  7. Verification Check — 고객 주장 중 운영자 확인 필요 항목 감지 (V1~V4). Draft와 함께 하이라이트 배너 표시 (§7.8 참조)

7.4 생성 파라미터

파라미터값설명

7.5 System Prompt 구조

Claude에게 전달되는 System Prompt는 다음 요소로 구성됩니다:

  • 역할 정의 (Toomics 전문 CS 상담원)
  • CS Manual Reference (RAG에서 추출한 관련 섹션, 최대 4,000자)
  • 응답 가이드라인 (7개 규칙)
  • 표준 인사말 / 맺음말 (마켓 언어별)

7.6 Fallback 응답

Claude 호출이 실패하면 템플릿 기반 기본 응답이 생성됩니다:

{greeting}

Thank you for your inquiry regarding {category}.
We are looking into your request and will get back to you shortly.

{closing}

7.7 Draft 편집 & Re-score

AI가 생성한 초안은 textarea에서 자유롭게 편집할 수 있습니다. 편집 후 Re-score 버튼을 눌러 수정된 텍스트의 품질 점수를 재계산할 수 있습니다.

7.8 운영자 확인 필요 (Verification Checker)

고객이 주장하는 내용 중 운영자가 사실 확인해야 하는 항목을 자동 감지합니다. 스팸/NO_MATCH와 달리 Draft는 정상 생성되며, 확인 필요 배너가 Draft 상단에 추가로 표시됩니다.

감지 패턴 (4개)

ID패턴감지 키워드확인 사항
V1 작품 서비스 상태 "discontinued", "removed", "taken down", "no longer available", "can't read purchased" 해당 작품의 실제 서비스 상태 확인 (작품명 자동 추출)
V2 결제 내역 "charged twice", "double charge", "unauthorized charge", "didn't authorize", "payment error" 관리자 패널에서 실제 결제 내역 확인
V3 계정 상태 "account suspended", "account blocked", "can't access account", "locked out" 계정 상태 및 제재 이력 확인
V4 VIP 구독 상태 "paid but no VIP", "subscription not active", "still asked to subscribe", "bought VIP but can't access" VIP 구독 상태 관리자 패널에서 확인

배너 표시 예시

ℹ️ 운영자 확인 필요 V1 — 작품 서비스 상태: 고객이 "Sister-in-law" 작품이 서비스 중단/삭제되었다고 주장합니다. 관리자 패널에서 해당 작품의 실제 서비스 상태를 확인해주세요.

스팸/NO_MATCH와의 차이

구분SpamNO_MATCHVerification
Draft 생성❌ 차단❌ 차단✅ 정상 생성
배너 위치Draft 대체Draft 대체Draft 상단 (파란색)
상담원 액션Mark as SpamEscalate확인 후 답변 발송

워크플로우

  1. AI Draft 생성 시 고객 텍스트에서 V1~V4 패턴 자동 스캔
  2. 매칭 패턴이 있으면 파란색 ℹ️ 확인 배너 + 정상 Draft 동시 표시
  3. 상담원이 관리자 패널에서 해당 사항을 확인
  4. 확인 결과를 반영하여 Draft 수정 후 Reply & Pending/Solved

9. Quality Scoring System

9.1 6차원 차등 가중치 채점

각 차원 0.0~1.0 점, 차등 가중치로 총점 산출. Safety에 최고 가중치(30%)를 부여하여 안전성을 최우선합니다.

차원가중치채점 기준
Template Match 20% 매뉴얼 템플릿 사용 여부: 사용=1.0, 미사용=0.3, Fallback=0.5
Completeness 15% 인사말(0.33) + 본문 80자+(0.34) + 맺음말(0.33) 유무
Safety 30% 금지 패턴 감지: 위반당 -0.25 (최소 0.0). 최고 가중치
Length 10% 100~2,000자 최적 (1.0), 미달/초과 시 감점
Accuracy 15% 매뉴얼 내용과의 일치도. 소스에 없는 문구 포함 시 즉시 0점
Tone 10% 마켓별 톤 가이드 준수 (FR: vous, DE: Sie, JP: 敬語 등)

10. Spam/Phishing Detection

10.1 감지 패턴 (9개 그룹, P1~P9)

#패턴감지 대상예시
P1 법률사무소 사칭 유명 로펌 이름 + DMCA/저작권 키워드 + 첨부파일 유도 "Mayer Brown LLP", "DMCA Takedown", "Evidence_Record"
P2 랜덤 해시 문자열 32자+ hex 또는 28자+ 알파뉴메릭 랜덤 토큰 "huidjgxogdaeh33ibn9srp..."
P3 외부 링크/다운로드 유도 위험 파일 확장자 URL, URL 단축기, 다운로드 유도 문구 "bit.ly/...", ".exe", "click here to download"
P4 긴급성 압박 "urgent action required" + 짧은 기한 "within 48 hours", "legal action will be taken"
P5 파트너십 사칭 "partnership program", "creator partnership", "exclusive offer" "selected creators for CapCut partnership"
P6 가짜 구독 해제 about:{LINK}, {UNSUBSCRIBE_LINK} 미치환 변수 "about:{unsubscribe}"
P7 외부 서비스 홍보 CapCut Pro, Netflix free, TikTok premium 등 "Get CapCut Pro free for creators"
P8 외부 이메일 응답 reply@non-toomics-domain.com 형태 "reply@marketing-platform.inc"
P9 소셜미디어 사칭 "Facebook/Meta policy violation" + "permanently disabled" + "appeal" "Your page will be permanently disabled. Submit an appeal here"

10.2 판정 기준

Spam 판정 조건 2개 이상의 패턴 그룹이 매칭되면 스팸으로 판정합니다. 단, P1(법률사무소) + 첨부파일 유도 패턴이 동시 매칭되면 단독으로도 스팸 판정됩니다.

10.3 스팸 감지 시 동작

  1. AI Draft 생성 중단 (Claude 호출하지 않음)
  2. SPAM WARNING 경고 배너 표시
  3. 감지 사유 목록 표시 (어떤 패턴이 매칭되었는지)
  4. Mark as Spam 버튼 제공

10.4 Mark as Spam 처리

"Mark as Spam" 버튼 클릭 시 Zendesk API로 아래 동작이 실행됩니다:

  1. 확인 다이얼로그 (실수 방지)
  2. PUT /tickets/{id}/mark_as_spam.json — 티켓 삭제 + 요청자(requester) 계정 정지
Warning: 비가역적 동작 Mark as Spam은 티켓을 영구 삭제하고 해당 고객의 Zendesk 계정을 정지합니다. 오판 시 복구가 어려우므로, 확실한 스팸에만 사용해야 합니다. 불확실한 경우 Escalate로 팀 리드에게 전달하세요.

10.5 티켓 전환 시 초기화

다른 티켓을 선택하면 스팸 배너, Draft 텍스트, Quality 점수 등이 모두 리셋됩니다.

12.4 Rate Limit 처리

Zendesk 429 응답 시 Retry-After 헤더 값만큼 대기 후 자동 재시도합니다. 추가로 연결 오류/타임아웃 시 exponential backoff (1~10s, 최대 3회)로 재시도합니다.

12.5 카테고리 분류 (4단계 + 후보정)

티켓의 메타데이터와 콘텐츠를 기반으로 4단계 우선순위 분류 후, 내용 기반 후보정(reclassification)을 수행합니다:

단계소스설명신뢰도
1. custom_fieldZendesk 커스텀 필드고객이 접수 시 선택한 카테고리중간
2. intent_tagZendesk AI 인텐트 태그intent__billing__refund 등 자동 태깅높음
3. bare_tagZendesk 일반 태그subscription, refund 등중간
4. keyword제목/본문 키워드"cancel subscription" 등 텍스트 매칭낮음

12.5.1 내용 기반 후보정 (Reclassification)

분류 후 subject + customer_inquiry 내용을 분석하여 오분류를 자동 교정합니다 (9개 언어 지원):

  • intent_tag 분류는 건너뜀 — AI 인텐트 태그의 신뢰도가 높으므로 후보정 제외
  • 허용된 전환 쌍만 교정 — 예: Payment→Refund(허용), Account→IP(불허)
  • 보수적 규칙 — 원 카테고리에도 시그널이 있으면 새 카테고리가 2배 점수 필요
  • 교정률: 약 3.8% (67건/1,779 — 2026년 2월 데이터 기준)

13. Operational Policies

13.1 AI Draft 사용 정책

  • AI Draft는 참고용 초안입니다. 반드시 상담원이 내용을 검토하고 필요 시 수정한 후 발송해야 합니다.
  • AI 생성 텍스트를 무검토 발송하는 것은 금지합니다.
  • 품질 점수 60% 미만인 Draft는 전면 재작성을 권장합니다.
  • Safety 위반(경쟁사 언급, 내부 정보 노출 등)이 있는 Draft는 해당 부분을 반드시 제거/수정해야 합니다.

13.2 스팸 처리 정책

  • 스팸 감지 경고가 뜬 티켓은 반드시 내용을 확인합니다.
  • 확실한 스팸만 "Mark as Spam"으로 처리합니다.
  • 불확실한 경우 Escalate로 팀 리드에게 전달합니다.
  • 정상 고객 티켓을 스팸 처리하면 고객 계정이 정지되므로 주의가 필요합니다.

13.3 에스컬레이션 정책

  • Escalate 시 내부 메모에 에스컬레이션 사유를 반드시 기재합니다.
  • 에스컬레이션된 티켓은 팀 리드가 24시간 이내 확인해야 합니다.
  • 처리 완료 후 Reply & Solved로 마감합니다.

13.4 데이터 보안 정책

  • 고객 PII(이메일, 이름, 결제 정보)는 대시보드 외부로 복사/저장 금지
  • JWT 토큰은 sessionStorage에만 저장 (탭 닫으면 자동 삭제)
  • DASHBOARD_SECRET_KEY는 최소 32자의 랜덤 문자열 사용
  • 기본 비밀번호(admin/admin)로 운영 환경 접속 금지

13.5 응답 금지 사항

  • 경쟁사 이름 언급 (Webtoon, Tapas, Tappytoon, Lezhin)
  • 내부 시스템/프로세스 노출 ("internal note", "escalation process" 등)
  • en.toomics.com 대신 마켓별 도메인 사용 (예: fr.toomics.com)
  • 미치환 템플릿 변수 ({customer_name} 등) 발송
